# Russian Roulette Game Specifications

## Goal
Reactを使用したWebベースのロシアンルーレットゲームを作成する。
現段階では画像や音声を使用せず、ゲームの仕様（フェーズ移行、装填、発砲ロジック）の検証を目的とする。

## 仕様 (Game Specifications)

### 1. ゲームループとフェーズ
ゲームは以下のステート（状態）を持ちます。

*   **SELECTION (操作選択)**: プレイヤーが行動を選択する待機状態。
*   **LOADING (装填)**:
    *   誤操作防止のため、確認ステップを挟む。
    *   **アクション**: 確認ボタン押下後、スワイプ操作（PCではスライダーをドラッグして端まで動かす等）を行うことで、シリンダーが回転し装填される。
    *   この操作によりゲーム状態がリセットされ、新しいラウンドが開始される。
*   **FIRING (発砲)**: 実際に引き金を引くフェーズ。
*   **GAME_OVER (終了)**: 実弾が発射された状態。リセットが可能。

### 2. 銃の仕組み (Gun Logic)
*   6発装填可能なリボルバーを想定。
*   実弾(Live Round): 1発
*   空砲(Blank): 5発
*   **装填処理**:
    *   実弾の位置（0~5）をランダムに決定する。
    *   現在のシリンダー位置（Current Chamber）を0にリセットする（またはランダムに決定後、相対的に管理）。
*   **発砲処理**:
    *   現在のシリンダー位置に実弾があるか判定。
    *   判定後、シリンダーを次の位置に進める（`current index + 1`）。

### 3. UI/UXフロー
1.  **初期画面 (SELECTION)**
    *   ボタン: `[操作選択へ]` (※ゲーム開始時など)
    *   現在のステータス表示（"装填済み" / "未装填"）
    *   ボタン: `[発砲フェーズへ (Go to Fire)]`
    *   ボタン: `[再装填フェーズへ (Reload/Reset)]`
2.  **装填画面 (LOADING)**
    *   説明: "シリンダーを回して弾を込めてください"
    *   **UI**: スライダー（`<input type="range">`等を使用）
        *   ユーザーがスライダーを左から右へドラッグ（スワイプ動作を模倣）。
        *   最大値に達したら「カチャッ」という文字表示と共に装填処理実行 -> SELECTIONへ。
3.  **発砲画面 (FIRING)**
    *   ボタン: `[FIRE]` (発砲ボタン)
        *   **MouseDown**: 撃鉄を起こすなどの予備動作（今回はコンソールログまたはテキスト変化で表現）
        *   **MouseUp**: 発砲判定実行
    *   ボタン: `[戻る]` (SELECTIONへ戻る)
    *   フィードバック:
        *   空砲の場合: "Click... (Safe)" -> そのまま継続
        *   実弾の場合: "Game Over" -> GAME_OVER画面へ
3.  **ゲームオーバー画面**
    *   メッセージ: "Game Over"
    *   ボタン: `[リトライ]` -> 初期状態へリセット
